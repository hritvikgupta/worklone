# Plan: Remove Separate Planning LLM Call - Dynamic ReAct Plan Identification

## Problem
Currently there's a SEPARATE LLM call (`_plan_execution_mode()` at line 386-491 in `react_agent.py`) that classifies every user request as "direct", "multi_step", or "background" BEFORE the main ReAct loop runs.

## Goal
Have the ReAct agent dynamically determine if a request needs planning, integrated into its normal reasoning loop instead of a separate LLM call.

## Changes Required

### 1. Modify `backend/employee/react_agent.py`

**A. Remove `_plan_execution_mode()` method (lines 386-491)**
- Delete this entire method - it's the separate planning LLM call

**B. Update `chat()` method (line 558)**
- Remove the call to `planning_decision = await self._plan_execution_mode(message, llm_messages)`
- The agent will decide dynamically through normal ReAct reasoning

**C. Modify system prompt to guide the agent:**
In `GENERIC_EMPLOYEE_SYSTEM_PROMPT` (lines 80-97):
- Update the instruction to have the agent itself determine whether to:
  - Answer directly (for simple/one-off requests)
  - Create a plan first (for multi-step tasks requiring approval)
- Add explicit guidance for the agent to use `manage_tasks` + `ask_user` when it decides planning is needed

**D. Update the `chat()` flow:**
- Remove all `planning_decision` related logic (lines 558-711)
- The agent will naturally call `manage_tasks` with `create_plan` when it determines it's needed
- The frontend already handles `plan_created` events emitted by `manage_tasks` tool

### 2. No frontend changes needed
The frontend already properly handles:
- `plan_created` events from `manage_tasks` tool (line 593-604)
- `confirmation_required` for plan approval (line 718-765)
- The plan panel is displayed based on these events

## Implementation Order
1. Remove `_plan_execution_mode()` method
2. Remove planning-related code in `chat()` method (the if block checking `planning_decision.get("mode")`)
3. Update system prompt to guide agent self-determination
4. Test - the agent will naturally create plans when it decides via `manage_tasks`

## Verification
- Direct/simple requests → agent responds directly without plan
- Multi-step tasks → agent creates plan and triggers `ask_user` for approval
- Frontend shows plan panel as before
